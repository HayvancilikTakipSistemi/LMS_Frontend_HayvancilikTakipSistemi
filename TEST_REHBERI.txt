▌ GÜNLÜK KULLANIM — Kod değiştirdikten sonra GitHub'a kaydet
──────────────────────────────────────────────────────────────────────────────

KOMUT 1 — Neyin değiştiğini gör:
  git status

  NE YAPAR  → Değişen, eklenen, silinen dosyaları listeler.
  ÇIKTI     → "modified: LMS.Server/Program.cs" gibi satırlar

──────────────────────────────────────────────────────────────────────────────

KOMUT 2 — Değişiklikleri hazırla (stage'e al):
  git add .

  NE YAPAR  → Tüm değişen dosyaları commit'e hazırlar.
              Sadece belirli bir dosyayı eklemek istersen:
              git add LMS.Server/Program.cs

──────────────────────────────────────────────────────────────────────────────

KOMUT 3 — Değişiklikleri kaydet (commit):
  git commit -m "ne yaptığını buraya yaz"

  ÖRNEKLER:
    git commit -m "VeterinarianController guncellendi"
    git commit -m "Login sayfasi tasarimi duzeltildi"
    git commit -m "Sprint 2 tamamlandi"

  NE YAPAR  → Değişiklikleri LOCAL olarak kaydeder.
              Henüz GitHub'a gitmez, sadece bilgisayarında.

──────────────────────────────────────────────────────────────────────────────

KOMUT 4 — GitHub'a gönder (push):
  git push

  NE YAPAR  → Commit'leri GitHub remote reposuna yükler.
              Bu adımdan sonra ekip arkadaşların görebilir.

  BEKLENEN ÇIKTI:
    To https://github.com/HayvancilikTakipSistemi/LMS_Frontend_...
       ae861fa..xyz1234  main -> main

──────────────────────────────────────────────────────────────────────────────

HEPSİNİ BİRDEN (3 komut sırayla):
  git add .
  git commit -m "değişiklik açıklaması"
  git push


▌ ARKADAŞIN YENİ KOD YÜKLEDI — Bilgisayarına çek
──────────────────────────────────────────────────────────────────────────────

  git pull

  NE YAPAR  → GitHub'daki yeni kodları bilgisayarına indirir.
              Her oturuma başlarken bunu çalıştır.

  BEKLENEN ÇIKTI (yeni şeyler varsa):
    Updating ae861fa..xyz1234
    Fast-forward
     LMS.Server/Program.cs | 5 ++---

  BEKLENEN ÇIKTI (zaten güncel ise):
    Already up to date.
